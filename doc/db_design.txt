semester.db:
============

1. Table users:

   Associates e-mail addresses to a unique Autosub ID.
   In addition we want to know for each user, which task he is currently working on, the date + time when he first sent an e-mail and
   when he first successfully submitted the last task.

   CREATE TABLE Users(UserId INTEGER PRIMARY KEY AUTOINCREMENT, Name TEXT, email TEXT, first_mail INT, last_done INT, current_task INT)

   Table created by: Fetcher
   UserId: set upon entry creation -> CONSTANT
   Name: set upon entry creation -> CONSTANT
   email: set upon entry creation -> CONSTANT
   first_mail: set upon entry creation -> CONSTANT
   last_done:  initilized to NULL upon creation, set by worker thread
   current_task: updated by worker thread 

2. Task statistics:

   Logs for each tasks, how often a submission was done and how often the task was submitted successfully.
   
   CREATE TABLE Users(TaskId INTEGER PRIMARY KEY, nr_submissions INT, nr_successful INT)

   all maintained by the worker threads.

3. Autosub statistics:
   Contains a number of counters that count different events.
   
    - number of e-mails fetched --> Fetcher 
    - number of e-mails sent --> Sender
    - number of questions received --> Fetcher

   CREATE TABLE Users(CounterId INTEGER PRIMARY KEY AUTOINCREMENT, Name TEXT, value INT)

4. UserTasks: Map users to generated tasks.

   CREATE TABLE UserTasks(uniqeID INTEGER PRIMARY KEY AUTOINCREMENT, TaskNr INT, UserId INT, TaskParameters TEXT, TaskDescription TEXT, TaskAttachements TEXT);

5. UserWhiteList:
 CREATE TABLE UserWhiteList(ID INTEGER PRIMARY KEY AUTOINCREMENT, email TEXT);
